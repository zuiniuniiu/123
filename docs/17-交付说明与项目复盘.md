## 5. 如果再给 1-2 次课，最该补什么

1. **密码哈希存储** — 使用 `PasswordHasher` 替代明文存储，这是最大的安全缺口
2. **管理员多账号** — 增加管理员注册/管理功能
3. **操作日志** — 记录关键操作（预约、取消、标记完成、座位变更）到日志表
4. **分页** — 预约列表和座位列表增加分页（演示数据扩展时必需）
5. **全局异常处理** — 自定义错误页面 + 日志记录
6. **浏览器全覆盖** — 将 Firefox 和 WebKit 加入 Playwright 项目配置
7. **CI 集成** — 配置 GitHub Actions 自动 build + test

## 7. 最终演示顺序建议

### 7.1 快速启动（1 分钟）

```bash
dotnet run --project src\LibrarySeatReservation.Web
# 打开 http://localhost:5000
```

### 7.2 用户端演示（3 分钟）

1. 首页 → 查看"推荐空闲座位"卡片（3~5 个空闲座位）
2. 导航栏下拉切换账号（张三 → 李四 → 王五）
3. 点击"查看全部座位" → 区域筛选"三楼自习区" + 楼层"3F" → 重置
4. 点击空闲座位 A-01 → 查看时段（08-12 已满，14-18 / 18-21 空闲）
5. 选择空闲时段 → "提交预约" → P04 确认 → 跳转 P05 我的预约
6. 在 P05 查看新预约 → 点"取消"并确认 → 状态变为"已取消"

### 7.3 管理端演示（3 分钟）

1. 访问 `/Admin/Login` → 输入 admin / admin123
2. P07 预约管理 → 按状态筛选"已预约" → 点"标记完成"
3. 导航到 P08 座位管理 → 切换座位状态（空闲↔维护）→ 新增/编辑/删除
4. 导航到 P09 统计页 → 查看 3 张汇总卡片 + 时段分布表
5. 左上角"退出" → 返回登录页

### 7.4 跨端回流（1 分钟）

1. 管理员标记完成 → 用户端 P05 我的预约可见状态变更
2. 管理员维护座位 → 用户端 P02 座位列表/ P03 座位详情可见变化

### 7.5 自动化测试演示（2 分钟）

```bash
# 终端 1：启动应用
dotnet run --project src\LibrarySeatReservation.Web --urls http://localhost:5000

# 终端 2：运行 Playwright 测试
npx playwright test tests/smoke.spec.ts --reporter=list
# 预期输出：19 passed (30s)

# 运行烟雾测试
node scripts/smoke.mjs
# 预期输出：7/7 全部通过
```
