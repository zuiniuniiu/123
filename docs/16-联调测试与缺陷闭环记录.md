### 2.2 浏览器覆盖

| 浏览器 | Playwright 配置 | 用途 |
|--------|----------------|------|
| Microsoft Edge (系统) | `channel: 'msedge'` | 主链路 E2E（默认） |

- `playwright.config.js` 根目录，3 个 project（msedge / chromium / firefox）
- 默认 `headless: true`，`baseURL: http://localhost:5000`
- 失败时截图，首次失败重试保留 trace
- 报告输出 HTML 到 `test-reports/`

## 5. 运行方法

```bash
# 1. 启动项目（需要另一个终端）
dotnet run --project src\LibrarySeatReservation.Web

# 2. 运行全部测试（默认 chromium）
npm test

# 3. 运行指定测试集
npm run test:smoke    # 烟雾测试
npm run test:user     # 用户端 E2E
npm run test:admin    # 管理端 E2E
npm run test:sync     # 状态回流
npm run test:edge     # 边界异常
npm run test:all      # 全部 chromium 测试

# 4. 兼容性测试（chromium + firefox）
npm run test:compat

# 5. 一键烟雾测试（bat 脚本）
scripts\smoke-test.bat
```

## 7. 默认补足项 / 当前假设

| 补足项 | 选择 | 原因 |
|--------|------|------|
| 测试框架选择 | Playwright + msedge | 审计要求明确使用 `channel: 'msedge'`，避免下载额外浏览器 |
| 测试运行前提 | 需先启动 `dotnet run` | Playwright 需要真实后端响应 |
| Node.js 前提 | 视为已安装 | 审计要求不展开 Node.js 安装 |
| 兼容性测试范围 | Chromium + Firefox | 覆盖主流浏览器引擎，Edge 通过 msedge channel 覆盖 |
| 不包含 Safari/WebKit | 不做 | Windows 环境限制，WebKit 无系统安装版 |
| Bug 严重度分级 | P0=阻塞, P1=严重, P2=一般, P3=建议 | 按项目一贯口径 |
