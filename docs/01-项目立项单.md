# 图书馆座位预约系统 · 项目立项单

## 1. 项目基本信息

| 字段 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统 |
| 项目简称 | SeatBook |
| 技术栈 | ASP.NET Core MVC (.NET 8) + Razor 视图 + SQL Server LocalDB + EF Core + Bootstrap |
| 部署方式 | 本地运行，IIS Express / Kestrel |
| 代码托管 | GitHub 或 Gitee 仓库 |
| 开发周期 | 约 4 周（含文档、编码、验收） |

## 2. 项目背景

很多高校图书馆虽然提供了实体座位，但"占座""抢座"现象一直没法根治。同学们到了图书馆发现座位被书包占了，或者转半天找不到空位。目前校内没有一个轻量的线上工具来解决这个问题。

与其等学校信息中心统一开发，不如我们自己做一个最小可用版本。本期项目就是基于 ASP.NET Core MVC 做一套"图书馆座位预约系统"，覆盖普通同学查座位、预约、取消预约，以及管理员维护座位和查看统计的核心场景。

## 3. 目标用户

- **普通用户（学生）**：在校学生，需要快速找到空座位、预约座位、管理自己的预约记录。
- **管理员（图书馆老师 / 学生助理）**：需要维护座位信息、查看预约情况、了解座位使用统计。

## 4. 项目目标

- 完成一个可运行的 Web 应用，学生能浏览座位、提交预约、取消预约。
- 管理员能登录管理后台，管理座位、查看预约列表和基础统计。
- 不做花哨功能，核心目标是一个用户能走通"查座 → 选座 → 预约 → 查看预约 → 取消预约"的完整链路。

## 5. 本期必须完成内容

1. **用户端**
   - 首页（推荐座位 / 快速入口）
   - 座位列表页（按区域、楼层筛选，展示座位状态）
   - 座位详情页（座位信息 + 预约按钮）
   - 预约提交（选择日期时段 → 确认 → 提交）
   - 我的预约（查看自己所有预约，可取消）
   - 取消预约

2. **管理端**
   - 管理员登录（Session / Cookie 方式）
   - 预约管理（查看所有预约、手动处理）
   - 座位管理（CRUD 座位，标记座位状态）
   - 统计页面（简单图表或数据汇总，如今日预约数、各时段热度）

3. **通用**
   - 用户模拟登录（体验账号切换，不做注册/密码体系）
   - 数据库初始化（Seed Data，启动即有演示数据）
   - 页面统一风格（Bootstrap 布局）

## 6. 本期暂不开发内容

- 支付 / 押金
- 扫码签到 / 闸机联动
- 消息推送（短信、邮件、微信）
- 完整账号体系（注册、改密、邮箱验证）
- 多馆区 / 多楼层联动
- 黑名单 / 信用分
- 移动端 App / 小程序（仅做响应式页面）
- 实时座位图（图形化座位布局）

## 7. 页面规模与页面分布

预计总页面数：**10 ~ 12 页**

| 模块 | 页面 | 说明 |
|------|------|------|
| 用户端 | 首页 | 推荐座位、快速查座入口 |
| 用户端 | 座位列表 | 筛选 + 列表/卡片展示 |
| 用户端 | 座位详情 | 单个座位信息 + 预约 |
| 用户端 | 预约提交 | 选时段 → 确认提交 |
| 用户端 | 我的预约 | 当前和历史预约列表 |
| 管理端 | 登录页 | 管理员登录表单 |
| 管理端 | 预约管理 | 所有预约记录，可搜索/处理 |
| 管理端 | 座位管理 | 座位增删改查 |
| 管理端 | 统计页 | 数据汇总（柱状图或表格） |
| 通用 | 布局/导航 | _Layout.cshtml、菜单栏 |

共约 10 个功能页面，每个页面控制在 1 个 Razor View + 1 个 Controller Action（或少量 partial）。

## 8. 关键业务主流程

```
学生操作流程：
  首页 → 座位列表 → 查看座位详情 → 提交预约 → 跳转到"我的预约" → 可取消预约

管理员操作流程：
  登录页 → 管理后台首页 → 预约管理 / 座位管理 / 统计页
```

核心业务规则：
- 每个座位一个时段只能被预约一次。
- 预约后状态变为"已预约"，其他人不可再约该时段。
- 取消预约后恢复为"可预约"状态。
- 管理员可强制关闭某条预约或修改座位状态。

以下每条均为可演示、可验收的动作：

| # | 验收项 | 演示方式 |

## 10. 风险与假设

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 开发时间不足，页面做不完 | 无法交付全部功能 | MVP 优先保证"学生预约链路"跑通，管理端做最小版本 |
| 页面太多失去控制 | 代码混乱，难以维护 | 严格按 10~12 页规划，不去发散加页面 |
| SQL Server LocalDB 安装或配置失败 | 本地无法运行 | 准备好回退方案：改用 SQLite 或内存数据库 |
| EF Core 数据迁移报错 | 数据库启动失败 | 使用 EnsureCreated + Seed Data，跳过 Migration |
| AI 生成的代码不稳定或有大坑 | 功能跑不通 | 每完成一个功能节点立即手动测试，不攒到最后 |
| Bootstrap 样式适配耗时 | 页面粗糙 | 使用 Bootstrap 官方示例模板，不做自定义 UI |
| 预约时段冲突逻辑写错 | 多人约同一个座位 | 提前画出实体关系图，事务写入，单元测试覆盖 |
