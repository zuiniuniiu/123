## B. 发现的问题清单

### B-1 审计文档内部问题

| # | 位置 | 问题类型 | 描述 |
|---|------|----------|------|
| R01 | 审计 §五 | 计数口径不一致 | 正文写"18 条边界异常中有 17 条已对应"，但 E08 已说明在 _Layout 处理"不需单独卡"，分母应为 17 而非 18。建议统一口径：18 条异常需处理的 17 条已落卡，1 条属框架层默认行为。 |
| R02 | 审计 §六"待确认项" | 措辞模糊 | "远端仓库连接已验证通过"和".NET 8 SDK 本地可用性"这两条不应列为"待确认"。远端已验证推送成功应升级为通过项；SDK 可用性属于运行前提，每个开发人员自行确认，不应作为项目级门禁项。建议将 SDK 检查移出，或改为"需开发人员自行确认项"。 |
| R03 | 审计 §四 | P2 判定过宽松 | 审计将"Bootstrap 风格"列为 P2 并判断"合理"。但 Bootstrap 是技术栈基础组件，所有页面依赖它渲染。严格说不应打分。建议改为"技术栈默认能力，不参与优先级评定"。 |
