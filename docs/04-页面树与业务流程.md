# 图书馆座位预约系统 · 页面树与业务流程

## 1. 页面总数与页面分布

本期共计 **9 个功能页面**，分为两大模块：

| 模块 | 页面数 | 页面列表 |
|------|--------|----------|
| 用户端 | 5 页 | 用户首页、座位列表页、座位详情页、预约提交页、我的预约页 |
| 管理端 | 4 页 | 管理员登录页、预约管理页、座位管理页、统计页 |
| **合计** | **9 页** | — |

此外包含 1 个共享布局 `_Layout.cshtml`，所有页面共用。

## 2. 用户端页面树

```
▼ 用户端（共享布局顶部导航 + 体验账号切换）
│
├── [P01] 用户首页（/Home/Index）
│   ├── 推荐座位卡片（3~5 个空闲座位）
│   └── "查看全部座位" 入口 → P02
│
├── [P02] 座位列表页（/Seat/Index）
│   ├── 区域/楼层筛选下拉框
│   ├── 座位卡片列表（编号、区域、状态）
│   │   └── 点击某卡片 → P03
│   └── 分页：本期不做（数据量小，全部展示）
│
├── [P03] 座位详情页（/Seat/Detail/{id}）
│   ├── 座位信息展示区（编号、区域、楼层、描述、状态）
│   ├── 预约操作区
│   │   ├── 日期选择器
│   │   ├── 时段选择（08:00-12:00 / 14:00-18:00 / 18:00-21:00）
│   │   └── "提交预约" 按钮 → 提交成功 → P05
│   └── 返回链接 → P02
│
├── [P04] 预约提交页（/Reservation/Create）
│   ├── 表单：座位号（只读）+ 日期 + 时段
│   ├── 提交 → 冲突检测 → 成功 → P05 / 失败 → 本页提示
│   └── 返回链接 → P02 或 P03
│
└── [P05] 我的预约页（/Reservation/MyReservations）
    ├── 预约记录列表（座位号、区域、日期、时段、状态）
    ├── 状态标签（已预约 / 已完成 / 已取消）
    ├── 已预约记录 → "取消" 按钮 → 确认 → 状态更新
    └── "继续预约" 入口 → P02
```

**页面间跳转汇总（用户端）：**

```
P01 ──→ P02 ──→ P03 ──→ P04 ──→ P05
 ^       ↑        ↑                 │
 │       │        └── 返回 ──────────┤
 │       │                          │
 │       └──────── 返回 ─────────────┤
 │                                   │
 └────────── "继续预约" ─────────────┘
```

## 3. 管理端页面树

```
▼ 管理端（需登录，Session 校验）
│
├── [P06] 管理员登录页（/Admin/Login）
│   ├── 用户名输入框
│   ├── 密码输入框
│   ├── "登录" 按钮 → 验证成功 → P07 或 P08/P09
│   └── 验证失败 → 本页显示错误提示
│
├── [P07] 预约管理页（/Admin/Reservation）
│   ├── 状态筛选（全部 / 已预约 / 已完成 / 已取消）
│   ├── 预约列表表格（ID、用户、座位号、日期、时段、状态）
│   └── 操作：已预约 → 标记为已完成
│
├── [P08] 座位管理页（/Admin/Seat）
│   ├── 座位列表表格（编号、区域、楼层、状态）
│   ├── "新增座位" 按钮 → 弹窗或跳转新增表单
│   ├── 编辑按钮 → 弹窗或跳转编辑表单
│   ├── 删除按钮 → 确认 → 删除（无关联预约时）
│   └── 状态切换按钮（空闲 ↔ 维护中）
│
└── [P09] 统计页（/Admin/Statistics）
    ├── 汇总卡片：座位总数 / 今日预约数 / 本月预约数
    └── 时段分布表格（时段名 → 预约次数）
```

**页面间跳转汇总（管理端）：**

```
[未登录]
  │
  ▼
P06 ──→ 登录成功 ──→ 管理后台首页（默认 P07 或 Dashboard）
                          │
                    ┌─────┼─────┐
                    ▼     ▼     ▼
                   P07   P08   P09
                    │     │     │
                    └─────┼─────┘
                          │
                    [退出登录] → P06

 导航栏：三个管理页面之间可随时互相跳转
 未登录访问 P07/P08/P09 → 自动跳回 P06
```

## 4. 用户主链路

### 4.1 正向预约链路（闭环）

```
[起点] 切换体验账号
   │
   ▼
P01 用户首页 ──→ P02 座位列表页
                      │
                      ▼ 选择一个空闲座位
                      │
                   P03 座位详情页
                      │
                      ▼ 选择日期 + 时段
                      │
                   P04 预约提交页
                      │
                      ▼ 冲突检测通过
                      │
                   P05 我的预约页（新记录出现）
                      │
                      ▼ 可在此取消
                      │
                   [终点：闭环] 用户可随时回到首页重新开始
```

### 4.2 取消预约链路

```
                    ┌──────────────────────────────────────┐
                    ▼                                      │
P05 我的预约页 → 找到"已预约"记录 → 点击取消 → 确认弹窗  │
                    │                                      │
                    ▼ 确认                                 │
            状态更新为"已取消"                              │
            对应座位恢复"空闲"                              │
                    │                                      │
                    └──── 继续浏览/预约 ──────────────────┘
```

### 4.3 用户端所有路径一览

| 路径 | 页面序列 | 是否闭环 |
|------|----------|----------|
| 主路径：浏览 → 预约 → 查看 | P01 → P02 → P03 → P04 → P05 | ✅ |
| 取消预约 | P05 → 取消操作 → P05（刷新） | ✅ |
| 从我的预约继续预约 | P05 → P02 → ... | ✅ |
| 首页快速选座 | P01 → P03（通过推荐卡片直接进入） | ✅ |
| 筛选后预约 | P01 → P02（筛选） → P03 → P04 → P05 | ✅ |

## 5. 管理主链路

### 5.1 管理员操作链路

```
[起点] 访问 /Admin/Login
   │
   ▼ 输入 admin / admin123
   │
P06 登录成功 → 跳转管理后台
   │
   ├──→ P07 预约管理：查看所有预约 → 标记已预约为已完成
   │
   ├──→ P08 座位管理：新增/编辑/删除座位 → 切换座位状态
   │
   ├──→ P09 统计页：查看汇总数据
   │
   └──→ [退出登录] → 跳回 P06
```

### 5.2 管理端所有路径一览

| 路径 | 页面序列 | 是否闭环 |
|------|----------|----------|
| 管理员登录 | P06 → P07/P08/P09 | ✅ |
| 未登录被拦截 | 访问 P07 → 跳转 P06 → 登录 → P07 | ✅ |
| 退出登录 | 任意后台页 → 退出 → P06 | ✅ |
| 跨页面导航 | P07 ↔ P08 ↔ P09 随时切换 | ✅ |

## 6. 状态流转

### 6.1 座位状态流转

```
        ┌─────────────────────────────────────┐
        │                                     │
        ▼          用户预约                    │
  ┌─────────┐ ──────────────────→ ┌─────────┐ │
  │  空闲   │                     │ 已预约  │ │
  │ (Avail) │ ←────────────────── │ (Booked)│ │
  └─────────┘    用户取消预约      └─────────┘ │
        │                                     │
        │ 管理员标记                           │
        ▼                                     │
  ┌─────────┐                                 │
  │ 维护中  │ ←──── 管理员也可直接修改 ────────┘
  │(Mainted)│
  └─────────┘
        │
        └── 管理员恢复空闲 ──→ [空闲]
```

**触发条件表：**

| 当前状态 | 触发动作 | 目标状态 | 操作者 |
|----------|----------|----------|--------|
| 空闲 | 用户提交预约 | 已预约 | 用户 |
| 已预约 | 用户取消预约 | 空闲 | 用户 |
| 已预约 | 管理员标记为已完成 | 已预约 | 管理员 |
| 空闲 | 管理员标记 | 维护中 | 管理员 |
| 已预约 | 管理员标记 | 维护中 | 管理员 |
| 维护中 | 管理员标记 | 空闲 | 管理员 |

### 6.2 预约状态流转

```
                 管理标记已完成
         ┌──────────────────────────────┐
         │                              │
         ▼                              │
  ┌──────────┐                     ┌──────────┐
  │ 已预约   │                     │ 已完成   │
  │ (Active) │                     │(Completed)│
  └──────────┘                     └──────────┘
         │
         │ 用户或管理员取消
         ▼
  ┌──────────┐
  │ 已取消   │
  │(Cancelled)│
  └──────────┘
```

**触发条件表：**

| 当前状态 | 触发动作 | 目标状态 | 操作者 |
|----------|----------|----------|--------|
| 已预约 | 用户点击取消 | 已取消 | 用户 |
| 已预约 | 管理员标记已完成 | 已完成 | 管理员 |
| 已预约 | 管理员标记已取消 | 已取消 | 管理员 |

### 6.3 座位状态与预约状态联动规则

| 场景 | 座位状态 | 预约状态 | 说明 |
|------|----------|----------|------|
| 座位空闲可约 | 空闲 | — | 无关联预约 |
| 用户预约成功 | 已预约 | 已预约 | 座位和预约同步锁定 |
| 用户取消预约 | 空闲 | 已取消 | 座位释放 |
| 管理员完成预约 | 已预约 | 已完成 | 座位保持已预约（历史记录） |
| 管理员标记维护 | 维护中 | 不变 | 预约不自动取消，但新预约被阻止 |
| 管理员恢复空闲 | 空闲 | 不变 | 座位恢复可约 |

## 7. 页面与流程关系说明

| 页面编号 | 页面名称 | 参与流程 | 页面职责 |
|----------|----------|----------|----------|
| P01 | 用户首页 | 正向预约链路入口 | 引导用户进入系统，推荐座位 |
| P02 | 座位列表页 | 正向预约链路 | 展示所有座位，提供筛选，是座位浏览的核心入口 |
| P03 | 座位详情页 | 正向预约链路 | 展示座位完整信息，是预约的前置信息页 |
| P04 | 预约提交页 | 正向预约链路 | 表单提交 + 冲突检测，是核心业务执行页 |
| P05 | 我的预约页 | 正向预约链路终点 + 取消预约链路 | 展示预约结果，提供取消操作 |
| P06 | 管理员登录页 | 管理端入口 | Session 登录验证，拦截未授权访问 |
| P07 | 预约管理页 | 管理端核心业务 | 查看和处理所有预约 |
| P08 | 座位管理页 | 管理端核心业务 | 座位的增删改查和状态维护 |
| P09 | 统计页 | 管理端辅助业务 | 数据汇总展示，辅助决策 |

**关键约束：**
- P01 ~ P05 构成用户端完整闭环，缺一不可
- P06 是所有管理端页面的唯一入口，未登录时 P07/P08/P09 不可访问
- 所有页面的导航栏统一由 `_Layout.cshtml` 维护，保持一致的页面跳转体验

## 8. 当前范围内页面

| 序号 | 页面 | 路由 | Controller | View 文件名 | 备注 |
|------|------|------|------------|-------------|------|
| P01 | 用户首页 | / | HomeController.Index | Index.cshtml | 默认路由 |
| P02 | 座位列表页 | /Seat | SeatController.Index | Index.cshtml | 含筛选表单 |
| P03 | 座位详情页 | /Seat/Detail/{id} | SeatController.Detail | Detail.cshtml | 需 id 参数 |
| P04 | 预约提交页 | /Reservation/Create | ReservationController.Create | Create.cshtml | POST 提交 |
| P05 | 我的预约页 | /Reservation/MyReservations | ReservationController.MyReservations | MyReservations.cshtml | 需登录态 |
| P06 | 管理员登录页 | /Admin/Login | AdminController.Login | Login.cshtml | GET/POST |
| P07 | 预约管理页 | /Admin/Reservation | AdminController.Reservation | Reservation.cshtml | 需 Session |
| P08 | 座位管理页 | /Admin/Seat | AdminController.Seat | Seat.cshtml | 需 Session |
| P09 | 统计页 | /Admin/Statistics | AdminController.Statistics | Statistics.cshtml | 需 Session |

**共享布局文件：** `/Views/Shared/_Layout.cshtml`

**Controller 组织方案：**

```
Controllers/
├── HomeController.cs        # P01 首页
├── SeatController.cs        # P02 列表, P03 详情
├── ReservationController.cs # P04 提交, P05 我的预约, 取消操作
└── AdminController.cs       # P06 登录, P07 预约管理, P08 座位管理, P09 统计
```

> 共 4 个 Controller，职责清晰，不拆分过细也不堆砌。

## 9. 暂不扩展页面

以下页面在本次 MVP 中明确 **不开发**，不进入页面树：

| 页面 | 原因 | 替代方案 |
|------|------|----------|
| 用户注册页 | 体验账号已覆盖 | 导航栏下拉切换账号 |
| 用户登录页（密码式） | 体验账号已覆盖 | 下拉切换，无密码 |
| 忘记密码页 | 无账号体系 | 不需要 |
| 座位图形化布局页 | 前端复杂度高 | 列表/卡片展示 |
| 预约签到页 | 依赖扫码硬件 | 不做 |
| 个人中心页 | 功能简单不需要 | 我的预约已覆盖所需信息 |
| 消息通知页 | 无推送能力 | 不做 |
| 操作日志页 | 课堂项目暂不需要 | 如需要可在统计页追加简单记录 |
| 多馆区切换页 | 范围已锁定单馆 | 不做 |
| 数据导出页 | 统计页数据已可见 | 查看统计页即可 |
