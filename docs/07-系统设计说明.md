# 图书馆座位预约系统 · 系统设计说明

## 1. 系统设计目标

- 用 ASP.NET Core MVC (.NET 8) 标准分层搭建，每个开发人员拿到文档就能知道代码往哪放
- 4 个 Controller 对应 9 个页面，不拆分过细也不堆砌
- Service 层收拢所有业务规则（冲突检测、状态联动），Controller 只做路由和视图调度
- EF Core 直接通过 DbContext 操作数据库，不加 Repository 层（项目规模小，没必要）
- 项目跑起来就能用，Seed Data 让演示数据自动初始化

## 2. 技术栈说明

| 层面 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | ASP.NET Core MVC (.NET 8) | 使用 `dotnet new mvc` 模板创建 |
| 视图 | Razor (.cshtml) | 服务端渲染，不前后端分离 |
| 数据库 | SQL Server LocalDB | 附加 `(localdb)\MSSQLLocalDB`，无需额外安装服务 |
| ORM | EF Core 8 + SQL Server Provider | `Microsoft.EntityFrameworkCore.SqlServer` |
| 前端 | Bootstrap 5.3 | CDN 引入，通过 `_Layout.cshtml` 全局加载 |
| 样式 | `seatbook.css` | 原型阶段的自定义样式直接复用（约 350 行） |
| 登录态 | Session（管理端）+ Cookie（用户端） | 内置 `AddSession()` + `AddCookie()` |

**降级方案：** 如果 LocalDB 配置失败，改用 SQLite 内存数据库，仅需改两行代码（Provider 替换 + ConnectionString 修改）。

**目录设计原则：**
- 不放 `Services/Interfaces/` 三层子目录，接口和实现放同一文件（`.cs` 同一文件定义 interface + class，或用 partial 区分）
- 不放 `Repositories/` 层 —— 直接通过 Service 调用 DbContext
- `ViewModels/` 仅放页面专用的数据类，不放 Entity
- `wwwroot/` 下直接复用原型阶段的两个文件，不再重写

## 4. 分层说明

### 4.1 Entity 层（Models/Entities/）

职责：定义数据库表结构，字段与 PRD-Lite 保持一致。

| 文件 | 对应表 | 关键字段 |
|------|--------|----------|
| `SimulatedUser.cs` | SimulatedUsers | Id, Name, DisplayOrder |
| `Seat.cs` | Seats | Id, SeatNumber, Area, Floor, Description, Status |
| `Reservation.cs` | Reservations | Id, SeatId, UserName, ReservationDate, TimeSlot, Status, CreatedAt |
| `Admin.cs` | Admins | Id, Username, Password |

**关键约束：**
- Seat.Status 用 `int` 枚举：0=空闲, 1=已预约, 2=维护中
- Reservation.Status 用 `int` 枚举：0=已预约, 1=已完成, 2=已取消
- 同一 SeatId + ReservationDate + TimeSlot + Status=0 不可重复（唯一索引或用业务代码保证）

### 4.2 ViewModel 层（Models/ViewModels/）

职责：每个页面一个 ViewModel，只包含该页需要展示和提交的数据。不放 Entity 直接进 View。

| ViewModel | 给哪些页面用 | 包含内容 |
|-----------|-------------|----------|
| `SeatListViewModel` | P02 座位列表 | AreaFilter 选项列表, Seats 列表（含状态） |
| `SeatDetailViewModel` | P03 座位详情 | Seat 信息, 可约时段列表 |
| `ReservationCreateViewModel` | P04 预约提交 | SeatId（只读）, SeatNumber, Date, TimeSlot, UserName |
| `MyReservationsViewModel` | P05 我的预约 | ReservationItem 列表（含座位信息 + 状态） |
| `AdminLoginViewModel` | P06 管理员登录 | Username, Password |
| `StatisticsViewModel` | P09 统计页 | TotalSeats, TodayReservations, MonthReservations, TimeSlotDistribution |

### 4.3 Service 层（Services/）

职责：所有业务逻辑放在这里。Controller 调用 Service，Service 调用 DbContext。

**SeatService：**

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `GetAllAsync(areaFilter)` | string? areaFilter | List<Seat> | 获取座位列表，支持按区域筛选 |
| `GetByIdAsync(id)` | int id | Seat? | 获取单个座位 |
| `GetAvailableSeatsAsync(count)` | int count | List<Seat> | 获取前 N 个空闲座位（首页推荐） |
| `GetAreasAsync()` | 无 | List<string> | 获取所有区域列表（筛选下拉用） |

**ReservationService：**

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `CreateAsync(seatId, userName, date, timeSlot)` | 字段 | (bool success, string message) | 提交预约，内含冲突检测 |
| `GetMyReservationsAsync(userName)` | string | List<Reservation> + Seat 信息 | 按用户名查预约 |
| `CancelAsync(reservationId, userName)` | int, string | (bool success, string message) | 取消预约，释放座位 |
| `GetAllAsync(statusFilter)` | int? | List<Reservation> + Seat 信息 | 管理员查看全部 |
| `MarkCompletedAsync(reservationId)` | int | bool | 管理员标记完成 |

**AdminService：**

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `LoginAsync(username, password)` | string, string | bool | 验证管理员账号 |
| `GetAllSeatsAsync()` | 无 | List<Seat> | 获取全部座位（管理端） |
| `AddSeatAsync(seat)` | Seat | bool | 新增座位 |
| `UpdateSeatAsync(seat)` | Seat | bool | 编辑座位 |
| `DeleteSeatAsync(seatId)` | int | (bool success, string message) | 删除座位（检查关联预约） |
| `ToggleSeatStatusAsync(seatId)` | int | bool | 切换空闲↔维护中 |

**StatisticsService：**

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `GetStatisticsAsync()` | 无 | StatisticsViewModel | 聚合查询获取全部统计数字 |

### 4.4 Controller 层（Controllers/）

职责：接收请求 → 调用 Service → 组装 ViewModel → 返回 View。不做业务判断，只做调度和视图。

**每个 Action 的标准化流程：**
```
1. GET Action: 调用 Service 获取数据 → 组装 ViewModel → 返回 View(viewModel)
2. POST Action: 验证参数 → 调用 Service 执行业务 → 成功则 RedirectToAction / 失败则返回 View 带错误
```

**Controller 职责矩阵：**

| Controller | Action | HTTP 方法 | 路由 | 对应页面 |

| ReservationController | Cancel | POST | /Reservation/Cancel/{id} | P05（操作） |
| AdminController | Login | GET/POST | /Admin/Login | P06 |
| AdminController | Logout | POST | /Admin/Logout | P06（操作） |
| AdminController | Reservation | GET | /Admin/Reservation | P07 |
| AdminController | MarkCompleted | POST | /Admin/MarkCompleted/{id} | P07（操作） |
| AdminController | Seat | GET | /Admin/Seat | P08 |
| AdminController | AddSeat | POST | /Admin/AddSeat | P08（操作） |

### 4.5 Data 层（Data/）

**AppDbContext.cs：**
- 继承 `DbContext`
- 包含 `DbSet<Seat>`、`DbSet<Reservation>`、`DbSet<SimulatedUser>`、`DbSet<Admin>`
- `OnModelCreating` 中配置：表名、字段约束、唯一索引（SeatId + ReservationDate + TimeSlot + Status=0 的复合检查）
- 不做 Migration，使用 `EnsureCreated()`

**DbInitializer.cs：**
- 静态方法 `Seed(AppDbContext context)`
- 如果数据库无数据，插入：
  - 3 个体验账号：张三、李四、王五
  - 1 个管理员：admin / admin123
  - 8 个座位（分布在 3 个区域，包含 3 种状态）
  - 5 条示例预约（含不同状态）
- 在 `Program.cs` 中 `app` 启动后调用

## 7. 主链路工程落点

### 链路 1：学生预约（P01 → P02 → P03 → P04 → P05）

| 步骤 | 页面 | Controller/Action | Service 调用 | 数据说明 |
|------|------|-------------------|-------------|----------|
| 1 | 首页 | HomeController.Index | SeatService.GetAvailableSeatsAsync(5) | 取前 5 个空闲座位 |
| 2 | 座位列表 | SeatController.Index | SeatService.GetAllAsync(areaFilter) | 取全部座位，区域筛选 |
| 3 | 座位详情 | SeatController.Detail | SeatService.GetByIdAsync(id) | 单条座位 + 可约时段 |
| 4 | 提交预约 | ReservationController.Create (POST) | ReservationService.CreateAsync(...) | 事务写入，冲突检测 |
| 5 | 我的预约 | ReservationController.MyReservations | ReservationService.GetMyReservationsAsync(userName) | 按用户查预约 |

**关键代码位置：**
- 冲突检测逻辑在 `ReservationService.CreateAsync()` 内
- 座位状态联动（预约后改 Seat.Status=1）也在同一个 Service 方法内
- 使用 EF Core 事务：`using var transaction = await _context.Database.BeginTransactionAsync()`

### 链路 2：管理员后台（P06 → P07/P08/P09）

| 步骤 | 页面 | Controller/Action | Service 调用 | 数据说明 |
|------|------|-------------------|-------------|----------|
| 1 | 登录 | AdminController.Login (POST) | AdminService.LoginAsync(u, p) | Session 写入 |
| 2 | 预约管理 | AdminController.Reservation | ReservationService.GetAllAsync(filter) | 全部预约 |
| 3 | 座位管理 | AdminController.Seat | AdminService.GetAllSeatsAsync() | 全部座位 |
| 4 | 统计页 | AdminController.Statistics | StatisticsService.GetStatisticsAsync() | 聚合查询 |

## 8. 异常处理原则

### 8.1 异常分级

| 级别 | 示例 | 处理方式 |
|------|------|----------|
| 用户操作错误 | 预约冲突、未选时段、未登录 | Controller 中 `ModelState.AddModelError()` → View 显示红色提示 |
| 业务规则违反 | 删除有关联预约的座位 | Service 返回 `(false, "错误信息")` → Controller 处理提示 |
| 系统异常 | 数据库连接失败、空引用 | 全局 `app.UseExceptionHandler()` 统一跳转到错误页 |

### 8.2 具体处理策略

- **预约冲突**：Service 层用 `AnyAsync()` 检查是否存在同座位+同日期+同时段+状态=0 的记录 → 存在则返回失败
- **删除座位**：先查 Reservation 表是否有该 SeatId 的记录（不限状态）→ 有则拒绝删除
- **未登录访问后台**：在 AdminController 中用 `OnActionExecuting` 或自定义 `[AdminOnly]` Filter 检查 Session → 无则跳转 `/Admin/Login`
- **空数据**：View 中判断 ViewModel 的列表为 null/empty 时展示空状态 UI
- **404 场景**：座位 ID 不存在 → Controller 返回 `NotFound()` 或自定义 404 页
- **500 场景**：全局异常处理，`Program.cs` 中 `app.UseExceptionHandler("/Home/Error")`

### 8.3 不在本期处理的异常

- 并发写入冲突（高并发场景课堂数据量碰不到）
- SQL 注入（EF Core 参数化查询天然防护）
- XSS（Razor 默认 HTML 编码）

## 9. Session / Cookie 使用原则

### 9.1 用户端体验账号（Cookie）

| 项目 | 内容 |
|------|------|
| 存储方案 | `Cookie`（非 Session，因为不需要过期和加密） |
| Key 名称 | `"SeatBook_UserName"` |
| 设置时机 | 用户在导航栏下拉切换账号时，POST 请求到某个 Action 写入 |
| 读取时机 | 每个页面通过 `_Layout.cshtml` 或基类 Controller 读取 |
| 有效期 | 浏览器会话有效期（关闭浏览器即清除） |
| 未设置时 | 默认取第一个体验账号（张三） |

**实现方式：**
```csharp
// 切换账号 POST 到 /Account/Switch
Response.Cookies.Append("SeatBook_UserName", userName, new CookieOptions
{
    Expires = DateTime.Now.AddDays(1) // 或留空 = 会话级别
});
```

### 9.2 管理端登录（Session）

| 项目 | 内容 |
|------|------|
| 存储方案 | `Session` |
| Key 名称 | `"IsAdmin"` |
| 设置时机 | AdminController.Login (POST) 验证成功后 |
| 校验时机 | AdminController 的每个 Action 前（Filter 或 OnActionExecuting） |
| 有效期 | 默认 20 分钟无操作过期 |
| 退出时机 | AdminController.Logout (POST) 清除 |

**实现方式：**
```csharp
// Program.cs 注册
builder.Services.AddDistributedMemoryCache();
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromMinutes(20);
    options.Cookie.HttpOnly = true;
});

// 登录成功
HttpContext.Session.SetString("IsAdmin", "true");

// 自定义 Filter
public class AdminOnlyAttribute : ActionFilterAttribute
{
    public override void OnActionExecuting(ActionExecutingContext context)
    {
        if (context.HttpContext.Session.GetString("IsAdmin") != "true")
        {
            context.Result = new RedirectToActionResult("Login", "Admin", null);
        }
    }
}
```

## 10. 当前阶段实现边界

| 维度 | 本期实现 | 本期不实现 |
|------|----------|------------|
| Controller | 4 个 Controller，每个 Action 标准 GET/POST | 无 API Controller，无前后端分离 |
| Service | 4 个 Service 覆盖全部业务逻辑 | 不引入缓存、消息队列、定时任务 |
| DbContext | EnsureCreated + Seed Data | 不做 Migration，不做读写分离 |
| ViewModel | 每个页面 1 个 ViewModel | 不做 AutoMapper（手动映射，就 9 个页面） |
| 验证 | Controller 内 ModelState 验证 + Service 内业务校验 | 不做 FluentValidation |
| 前端 | Bootstrap 5.3 + seatbook.css（复用原型） | 不引入 Vue/React/jQuery，不做 SPA |
| 日志 | 不使用 | 不使用 |
| 测试 | 无 | 不做单元测试（课堂项目以验收演示为准） |
| 部署 | 本地 IIS Express / Kestrel | 不做 Docker，不做云部署 |
