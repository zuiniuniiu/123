# 图书馆座位预约系统 · 页面卡与 UI 规范

## 1. UI 规范

### 1.1 设计原则

- **简洁优先**：不做冗余装饰，信息层级清晰，操作路径最短
- **Bootstrap 标准**：全程使用 Bootstrap 5 内置组件，不引入第三方 UI 库
- **响应式**：桌面优先，兼顾平板，手机端仅保证可读不保证完整操作

### 1.2 色彩体系

| 用途 | 颜色 | Bootstrap Token |
|------|------|----------------|
| 主色（导航、按钮、链接） | `#0d6efd` | `var(--bs-primary)` |
| 成功（空闲、预约成功） | `#198754` | `var(--bs-success)` |
| 警告（维护中、即将过期） | `#ffc107` | `var(--bs-warning)` |
| 危险（取消、删除） | `#dc3545` | `var(--bs-danger)` |
| 背景（内容区） | `#f8f9fa` | `var(--bs-light)` |
| 文字主色 | `#212529` | `var(--bs-dark)` |

### 1.3 排版

| 元素 | 规格 |
|------|------|
| 正文字体 | `system-ui, -apple-system, "Segoe UI", Roboto` |
| 标题 | `<h1>` ~ `<h4>` 按层级使用，`h1` 仅用于首页 |
| 导航栏 | `navbar-light bg-light`，Bootstrap 默认导航 |
| 卡片标题 | `<h5 class="card-title">` |
| 表格表头 | `<thead class="table-light">` |

### 1.4 布局结构

**用户端布局：**
```
┌──────────────────────────────────────────┐
│  导航栏（Logo + 导航链接 + 体验账号切换）│
├──────────────────────────────────────────┤
│                                          │
│              页面内容区                    │
│        (container mt-4)                  │
│                                          │
├──────────────────────────────────────────┤
│  页脚（版权信息，简单一行）              │
└──────────────────────────────────────────┘
```

**管理端布局：**
```
┌──────────────────────────────────────────┐
│  管理端导航栏（标题 + 管理菜单 + 退出）  │
├──────────────────────────────────────────┤
│                                          │
│              页面内容区                    │
│        (container-fluid mt-3)            │
│                                          │
└──────────────────────────────────────────┘
```
管理端无页脚，管理端导航栏与用户端视觉区分（背景色不同，建议 `bg-dark text-white`）。

### 1.5 组件规范

| 组件 | 使用规范 |
|------|----------|
| 按钮 | 主要操作用 `btn-primary`，危险操作用 `btn-outline-danger`，次要操作用 `btn-outline-secondary` |
| 卡片 | 座位卡片统一用 `.card` + `.shadow-sm`，宽度 18rem，hover 时轻微上浮效果 |
| 表格 | 管理端数据展示统一用 `table table-striped table-hover` |
| 表单 | 输入框统一应用 `form-control`，下拉框统一应用 `form-select` |
| 提示 | 成功提示用 `.alert-success`，错误提示用 `.alert-danger` |
| 徽标 | 状态标签用 `.badge` + 对应状态色：空闲=success、已预约=primary、维护中=warning、已取消=secondary、已完成=info |

### 1.6 原型三层次说明

| 层次 | 定义 | 本阶段交付物 |
|------|------|-------------|
| L1 线框图 | 页面布局骨架，无色彩无真实内容，标注信息区域和操作按钮位置 | 本节每张页面卡中的"布局示意"即线框图 |
| L2 高保真 | 带真实色彩、真实内容、Bootstrap 组件的静态页面 | 基于 Bootstrap 示例模板搭建的 HTML 页面 |
| L3 可交互原型 | 带页面跳转、简单交互的高保真成品 | 本阶段不做（直接进入编码） |

### P01 — 用户首页

| 字段 | 内容 |
|------|------|
| 页面标题 | 首页 |
| 路由 | `/Home/Index` |
| 页面目的 | 吸引用户进入系统，推荐座位，提供快速查座入口 |
| 目标用户 | 学生 |

**页面信息层级：**
1. 欢迎语 + 系统简介（一句话）
2. 推荐座位卡片区（3~5 个空闲座位，每个座位显示编号、区域、状态="空闲"）

| 字段 | 内容 |
|------|------|
| 页面标题 | 座位详情 — {座位编号} |

| 目标用户 | 学生 |

**页面信息层级：**
1. 座位信息卡（编号、区域、楼层、描述、当前状态徽标）
2. 预约操作区（日期选择器 + 时段选择 radio/button-group + "提交预约"按钮）

| 异常 | 座位 ID 不存在 → 显示 "座位不存在" |

| 异常 | 数据加载失败 → 显示错误提示 |

## 4. 用户端与管理端布局对比

| 维度 | 用户端 | 管理端 |

| 操作复杂度 | 单条操作为主（查看、预约、取消） | 批量/管理操作为主（CRUD、筛选、标记） |
