## C. 必改项

### C1. 🔴 P0 — Program.cs 调用 `Migrate()` 但无迁移文件，运行时崩溃

**问题**：`Program.cs:46` 使用 `db.Database.Migrate()`（旧版为 `EnsureCreated()`），但 `src/LibrarySeatReservation.Web/Migrations/` 目录不存在，没有任何迁移文件。`Migrate()` 在无迁移时抛出 `InvalidOperationException`，导致应用无法启动。

**修复要求（二选一）**：

方案 A（推荐 — 保持迁移方案）：生成首次迁移
```bash
dotnet ef migrations add InitialCreate
```
方案 B（快速修复 — 与旧版本兼容）：改回 `EnsureCreated()`
```cs
db.Database.EnsureCreated();  // Program.cs:46
```

**影响**：若不修复，应用在 `dotnet run` 后直接崩溃，无法访问任何页面。**阻塞进入 Sprint 2。**

### C2. 🟡 P1 — `SeatStatus` 枚举值命名与 docs/08 不一致

**问题**：docs/08-数据库设计 §3.3 定义 `SeatStatus` 枚举为 `Available=0, Reserved=1, Maintenance=2`。但实际代码将 `Reserved` 改为 `Booked`（`Models/Entities/SeatStatus.cs:6`）。虽然功能等价（值 =1），但跨文档不一致，且如果存在旧数据库（旧版 SeatBook 的 `SeatStatus` 可能用 `Reserved` 值），将导致反序列化问题。

**修复要求**：将 `SeatStatus.cs` 中的 `Booked` 改回 `Reserved`，同步更新所有引用了 `Booked` 的代码文件：
- `Data/DbInitializer.cs`（第 22 行 `SeatStatus.Booked` → `SeatStatus.Reserved`；第 26 行同理）
- `Services/ReservationService.cs`（第 38 行 `SeatStatus.Booked` → `SeatStatus.Reserved`；第 69 行恢复时改为 `SeatStatus.Available` — 当前已正确）
- `Services/AdminService.cs`（第 59 行 `SeatStatus.Booked` → `SeatStatus.Reserved`）
- `Views/Seat/Index.cshtml`（第 35 行 `Booked` → `Reserved`）
- `Views/Seat/Detail.cshtml`（第 21 行）
- `Views/Admin/Seat.cshtml`（第 45、51 行）
