# 图书馆座位预约系统 · PRD Lite

## 1. 项目概述

| 字段 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统（SeatBook） |
| 技术栈 | ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap |
| 开发周期 | 4 周 |
| 交付形式 | GitHub / Gitee 仓库，含完整可运行源码 + 演示数据 |

图书馆占座问题是高校常态。本项目目标是开发一个轻量 Web 系统，让学生能在线查看空闲座位、预约座位、管理自己的预约；让管理员能维护座位信息、查看预约情况、获取基础统计数据。不做硬件对接、不做支付、不依赖第三方服务。

## 2. 项目目标

- 走通一条完整预约链路：查座 → 选座 → 预约 → 查看 → 取消
- 管理员能通过后台管理座位和预约
- 系统启动后开箱即用，自带演示数据
- 所有操作在浏览器内完成，无需安装额外软件

## 3. 目标用户

| 用户类型 | 代表 | 核心诉求 |
|----------|------|----------|
| 普通学生 | 张三（大二，非计算机专业） | 快速找到空座，预约，可取消 |
| 管理员 | 李老师（图书馆老师 / 学生助理） | 管座位、管预约、看统计 |

## 4. 使用场景

**场景一：学生考前预约座位**
> 张三考试周复习，打开系统切换到自己常用的体验账号，按楼层筛选查看空闲座位，选中靠窗座位提交预约，第二天直接入座。

**场景二：学生取消预约释放座位**
> 张三中午离开图书馆，想起下午的预约用不上了，登录系统在"我的预约"中取消该时段预约，座位自动释放。

**场景三：管理员维护座位信息**
> 管理员李老师发现有座位损坏，登录后台将该座位状态改为"维护中"，学生端该座位不再显示为可预约。

**场景四：管理员查看统计数据**
> 李老师想了解本月座位使用情况，登录后台查看统计页，看到每日预约数量和时段分布数据。

## 5. 用户端核心功能

### 5.1 体验账号切换
- 导航栏右侧提供下拉框，列出预置的体验账号（如"张三""李四""王五"）
- 选择后立即切换身份，无需密码
- 当前账号信息显示在导航栏
- **验收标准**：下拉选择账号后，页面刷新显示对应用户名，所有操作以该账号身份进行

### 5.2 首页
- 展示欢迎信息和系统简介
- 推荐 3~5 个当前空闲座位（直接取空闲列表前几条）
- 提供"查看全部座位"快捷入口
- **验收标准**：首页显示推荐座位列表，点击可进入座位列表或座位详情

### 5.3 座位列表
- 以卡片或表格形式展示所有座位
- 每个座位显示：座位编号、所在区域/楼层、状态（空闲/已预约/维护中）
- 支持按区域/楼层下拉筛选
- **验收标准**：页面展示 ≥ 5 个座位，状态标记清晰可见，筛选后列表正确变化

### 5.4 座位详情
- 展示座位完整信息：编号、区域、楼层、描述、当前状态
- 提供预约操作区：日期选择器 + 时段选择（如 08:00-12:00、14:00-18:00）+ 提交按钮
- 如果该座位当前时段已被约，显示"已预约"提示
- **验收标准**：进入详情页看到座位信息和预约表单，提交后跳转至我的预约

### 5.5 预约提交
- 选择日期（默认今天）和时段
- 提交前检查该座位该时段是否已被预约（冲突检测）
- 提交成功 → 跳转到"我的预约"页并显示成功提示
- 提交失败（冲突） → 显示提示信息
- **验收标准**：成功预约后可在"我的预约"中看到记录；尝试预约已被约的时段提示失败

### 5.6 我的预约
- 列表展示当前用户所有预约记录
- 每条记录显示：座位号、区域、日期、时段、状态（已预约/已取消/已完成）
- 按预约时间倒序排列
- **验收标准**：页面显示当前账号的所有预约，状态正确，排列顺序合理

### 5.7 取消预约
- 在"我的预约"列表中，对"已预约"状态的记录提供"取消"按钮
- 点击后弹出确认提示
- 确认后预约状态变为"已取消"，对应座位该时段恢复空闲
- **验收标准**：取消后列表状态更新，对应座位在座位列表中恢复为"空闲"

## 6. 管理端核心功能

### 6.1 管理员登录
- 访问 /Admin/Login 进入登录页
- 输入用户名和密码（预设：admin / admin123）
- 使用 Session 记录登录状态
- 未登录状态下访问管理端页面自动跳转回登录页
- **验收标准**：输入正确账号密码可登录；输入错误提示失败；未登录访问后台被拦截

### 6.2 预约管理
- 表格展示所有用户的预约记录
- 每行显示：预约 ID、用户、座位号、日期、时段、状态
- 支持按状态筛选（全部/已预约/已取消/已完成）
- 管理员可手动将"已预约"标记为"已完成"
- **验收标准**：能查看所有预约，筛选生效，状态修改后数据正确更新

### 6.3 座位管理
- 表格展示所有座位
- 每行显示：座位编号、区域、楼层、状态、操作按钮
- 支持新增座位（填写编号、区域、楼层、描述）
- 支持编辑座位（修改信息）
- 支持删除座位（无关联预约时可删除）
- 支持直接切换座位状态（空闲/维护中）
- **验收标准**：可完成座位的增删改查，状态修改后在用户端立即可见

### 6.4 统计页
- 展示汇总数据：
  - 座位总数
  - 今日预约数
  - 本月预约数
  - 各时段预约分布（以表格形式展示）
- 不做图表，仅用数字和表格
- **验收标准**：数字与数据库实际数据一致，各时段分布表格正确

## 8. 核心业务流程

```
【学生端主流程】
首页 → 座位列表 → 座位详情 → 提交预约 → 我的预约 → 取消预约

步骤说明：
1. 首页点击"查看全部座位"进入座位列表
2. 可按区域/楼层筛选缩小范围
3. 点击座位卡片进入详情
4. 选日期时段 → 点击预约 → 冲突检测通过 → 预约成功 → 跳转我的预约
5. 在我的预约列表中点取消 → 确认 → 状态更新

【管理端主流程】
管理员登录 → 管理后台（预约管理 / 座位管理 / 统计页）

步骤说明：
1. 访问 /Admin/Login → 输入账号密码 → 登录成功
2. 进入预约管理查看所有预约
3. 进入座位管理维护座位信息
4. 进入统计页查看汇总数据
```

## 12. 状态流转说明

### 座位状态

```
[空闲] ──→ 被预约 ──→ [已预约]
  ↑                        │
  │                        │ 取消预约
  │                        ↓
  │                    [空闲]
  │
  │ 管理员操作
  ├─────────────────────→ [维护中]
  ←─────────────────────┘
```

### 预约状态

```
                ──→ [已完成]（管理员手动标记）
                │
[已预约] ──────┼──→ [已取消]（用户或管理员操作）
                │
                └── 到达预约日期后保持"已预约"，不做自动过期
```

业务规则：
- 同一座位 + 同一日期 + 同时段，只能有一条状态为"已预约"的记录
- 取消预约后对应座位的状态恢复为"空闲"
- 座位为"维护中"时，用户端不可提交预约

## 14. 暂不开发内容

以下内容在本期 **不开发**，已在需求评审中确认：

- 支付 / 押金
- 扫码签到 / 闸机联动
- 消息推送（短信、邮件、微信）
- 完整账号体系（注册、改密、邮箱验证）
- 多馆区 / 多楼层联动
- 黑名单 / 信用分
- 实时座位图（图形化布局）
- 统计图表可视化（柱状图/饼图）
- 分页功能（演示数据量小）
- 移动端响应式适配

## 15. 风险与假设

| 风险 | 影响 | 应对 |
|------|------|------|
| 开发时间不足 | 页面做不完 | MVP 已砍掉所有非核心功能，优先保用户端主链路 |
| SQL Server LocalDB 本地环境问题 | 无法启动数据库 | 备选方案：改用 SQLite 或内存数据库 |
| EF Core 迁移报错 | 数据库初始化失败 | 使用 EnsureCreated + Seed Data，不做 Migration |
| 预约并发冲突 | 同一座位被重复预约 | 使用数据库事务 + 乐观锁检查，代码层做双重校验 |
| AI 生成代码不稳定 | 功能跑不通 | 每完成一个页面立即测试，不累积 Bug |
| Bootstrap 样式陌生 | 页面不好看 | 直接套 Bootstrap 官方示例模板，不做自定义 CSS |
| 页码失控膨胀 | 超出 9 页 | 严格锁定页面清单，新增需求一律砍掉 |
