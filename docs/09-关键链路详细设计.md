# 图书馆座位预约系统 · 关键链路详细设计

## 1. 关键链路选择说明

以前序文档（页面树与业务流程、系统设计说明、数据库设计）为准，本期共 **2 条关键链路**：

| 链路 | 名称 | 页面范围 | 角色 | 说明 |
|------|------|----------|------|------|
| 链路一 | 学生预约主链路 | P01 → P02 → P03 → P04 → P05 | 学生 | 核心业务，覆盖"查座→选座→预约→查看→取消"完整闭环 |
| 链路二 | 管理员处理链路 | P06 → P07 / P08 / P09 | 管理员 | 后台管理，覆盖登录→管预约→管座位→看统计 |

以下逐条展开，每步标注路由、Controller Action、Service 方法、数据库变化、状态流转和边界异常。

### 2.4 DataAccess 动作

| 步骤 | 数据操作 | EF Core 代码示意 |
|------|----------|-----------------|
| P01 首页 | 读 Seats | `_context.Seats.Where(s => s.Status == 0).Take(5)` |
| P02 列表 | 读 Seats | `_context.Seats.Where(s => area == null \|\| s.Area == area)` |
| P02 区域 | 读 Seats (DISTINCT) | `_context.Seats.Select(s => s.Area).Distinct()` |
| P03 详情 | 读单个 Seat | `_context.Seats.FindAsync(id)` |
| P04 提交 | 写 Reservation + 写 Seat | 见下方完整事务代码 |
| P05 列表 | 读 Reservations + JOIN Seat | `_context.Reservations.Include(r => r.Seat).Where(r => r.UserName == name)` |
| P05 取消 | 写 Reservation + 写 Seat | 更新 Status 字段 |

**P04 提交预约的完整 DataAccess 代码（ReservationService.CreateAsync）：**

```csharp
public async Task<(bool Success, string Message)> CreateAsync(int seatId, string userName, DateTime date, string timeSlot)
{
    // 1. 检查座位是否存在且空闲
    var seat = await _context.Seats.FindAsync(seatId);
    if (seat == null)
        return (false, "座位不存在");
    if (seat.Status != 0)
        return (false, seat.Status == 2 ? "该座位正在维护中" : "该座位已被预约");

    // 2. 检查时段冲突（数据层+应用层双重保障）
    var conflict = await _context.Reservations
        .AnyAsync(r => r.SeatId == seatId
                    && r.ReservationDate == date
                    && r.TimeSlot == timeSlot
                    && r.Status == 0);
    if (conflict)
        return (false, "该时段已被预约，请选择其他时段");

    // 3. 事务写入
    using var transaction = await _context.Database.BeginTransactionAsync();
    try
    {
        var reservation = new Reservation
        {
            SeatId = seatId,
            UserName = userName,
            ReservationDate = date,
            TimeSlot = timeSlot,
            Status = 0,       // 已预约
            CreatedAt = DateTime.Now
        };
        _context.Reservations.Add(reservation);
        seat.Status = 1;      // 已预约

        await _context.SaveChangesAsync();
        await transaction.CommitAsync();
        return (true, "预约成功");
    }
    catch (DbUpdateException ex) when (ex.InnerException is SqlException sqlEx && sqlEx.Number == 2601)
    {
        // 唯一索引冲突（并发场景兜底）
        await transaction.RollbackAsync();
        return (false, "该时段已被预约，请选择其他时段");
    }
    catch
    {
        await transaction.RollbackAsync();
        throw;
    }
}
```

**P05 取消预约的完整 DataAccess 代码（ReservationService.CancelAsync）：**

```csharp
public async Task<(bool Success, string Message)> CancelAsync(int reservationId, string userName)
{
    var reservation = await _context.Reservations
        .Include(r => r.Seat)
        .FirstOrDefaultAsync(r => r.Id == reservationId);

    if (reservation == null)
        return (false, "预约记录不存在");
    if (reservation.UserName != userName)
        return (false, "只能取消自己的预约");
    if (reservation.Status != 0)
        return (false, "当前状态不允许取消");

    using var transaction = await _context.Database.BeginTransactionAsync();
    try
    {
        reservation.Status = 2;       // 已取消
        reservation.Seat.Status = 0;  // 空闲
        await _context.SaveChangesAsync();
        await transaction.CommitAsync();
        return (true, "已取消预约");
    }
    catch
    {
        await transaction.RollbackAsync();
        throw;
    }
}
```

### 2.5 数据库变化

| 步骤 | 操作类型 | 表 | 变化内容 |
|------|----------|-----|----------|
| P01 | 读 | Seats | 无变化 |
| P02 | 读 | Seats | 无变化 |
| P03 | 读 | Seats | 无变化 |
| P04 提交成功 | 插入 + 更新 | Reservations + Seats | Reservation 新增 1 行 (Status=0)；对应 Seat.Status 改为 1 |
| P05 列表 | 读 | Reservations, Seats | 无变化 |
| P05 取消成功 | 更新 | Reservations + Seats | Reservation 的 Status 改为 2；对应 Seat.Status 改为 0 |

### 2.6 状态流转

```
座位状态（Seat.Status）变化：
  0(空闲) ──→ 1(已预约)   [P04 用户提交预约]
  1(已预约) ──→ 0(空闲)   [P05 用户取消预约]

预约状态（Reservation.Status）变化：
  无记录 ──→ 0(已预约)    [P04 用户提交预约]
  0(已预约) ──→ 2(已取消) [P05 用户取消预约]
```

### 2.7 异常与边界情况

| 编号 | 场景 | 触发条件 | 处理方式 | 提示文案 |
|------|------|----------|----------|----------|
| E01 | 座位不存在 | 访问 P03 时 id 无效 | Controller 返回 `NotFound()` | 404 页面 |
| E02 | 座位维护中 | 访问 P03 时 Seat.Status=2 | View 隐藏预约表单，显示提示 | "该座位正在维护中，暂不可预约" |
| E03 | 未选日期时段 | P04 POST 时 date 或 timeSlot 为空 | ModelState 验证失败，返回本页 | "请选择日期和时段" |
| E04 | 时段冲突 | P04 POST 时同座位+同日期+同时段已有 Status=0 的记录 | Service 返回 false，View 显示错误 | "该时段已被预约，请选择其他时段" |
| E05 | 并发冲突 | 两个人同时预约同一座位同一时段 | 唯一过滤索引抛 DbUpdateException，事务回滚 | "该时段已被预约，请选择其他时段" |
| E06 | 取消他人预约 | P05 POST Cancel 时 UserName 不匹配 | Service 校验拒绝 | "只能取消自己的预约" |
| E07 | 重复取消 | 对已取消或已完成的预约点取消 | Service 校验 Status != 0 拒绝 | "当前状态不允许取消" |
| E08 | 未登录（无 Cookie）访问 | P04/P05 页面当前用户为空 | _Layout 中默认取"张三"，避免空值 | — |
| E09 | 预约过去日期 | 选择今天之前的日期提交 | Service 层做日期校验 `date < DateTime.Today` | "不能预约过去的日期" |
| E10 | 空数据首页 | 无空闲座位 | View 显示"暂无推荐座位" + 仍显示"查看全部座位"入口 | "当前没有空闲座位推荐" |

{
    var reservation = await _context.Reservations.FindAsync(reservationId);

    reservation.Status = 1;  // 已完成

public async Task<(bool Success, string Message)> DeleteSeatAsync(int seatId)
{
    var hasReservations = await _context.Reservations.AnyAsync(r => r.SeatId == seatId);
    if (hasReservations)
        return (false, "该座位有关联预约记录，无法删除");

    var seat = await _context.Seats.FindAsync(seatId);
    if (seat == null)
        return (false, "座位不存在");

    _context.Seats.Remove(seat);
    await _context.SaveChangesAsync();
    return (true, "已删除");
}
```

public async Task<bool> ToggleSeatStatusAsync(int seatId)
{
    var seat = await _context.Seats.FindAsync(seatId);
    if (seat == null) return false;
    if (seat.Status == 1) return false;  // 已预约状态不可手动切换

    seat.Status = seat.Status == 0 ? 2 : 0;  // 空闲↔维护中
    await _context.SaveChangesAsync();
    return true;
}

输出：跳转 P06
```

### 3.3 Service 职责

**AdminService 在本链路中涉及的方法：**

| 方法 | 调用方 | 内部逻辑 |
|------|--------|----------|
| `LoginAsync(username, password)` | AdminController.Login | `_context.Admins.AnyAsync(a => a.Username == username && a.Password == password)` |
| `GetAllSeatsAsync()` | AdminController.Seat | `_context.Seats.OrderBy(s => s.Id).ToListAsync()` |
| `AddSeatAsync(seat)` | AdminController.AddSeat | `_context.Seats.Add(seat); _context.SaveChangesAsync()` |
| `UpdateSeatAsync(seat)` | AdminController.EditSeat | `_context.Seats.Update(seat); _context.SaveChangesAsync()` |
| `DeleteSeatAsync(seatId)` | AdminController.DeleteSeat | 先查有无关联预约，无则删除 |
| `ToggleSeatStatusAsync(seatId)` | AdminController.ToggleStatus | 空闲↔维护中 切换 |

**StatisticsService 在本链路中涉及的方法：**

| 方法 | 调用方 | 内部逻辑 |
|------|--------|----------|
| `GetStatisticsAsync()` | AdminController.Statistics | 4 条聚合查询（详见数据库设计第 8 章） |

### 3.4 DataAccess 动作

| 步骤 | 数据操作 | 说明 |
|------|----------|------|
| P06 登录 | 读 Admins | 验证用户名密码 |
| P07 列表 | 读 Reservations JOIN Seats | 全部预约，可按状态筛选 |
| P07 标记完成 | 更新 Reservations | Status 0→1，Seat.Status 不变化 |
| P08 列表 | 读 Seats | 全部座位 |
| P08 新增 | 写 Seats | INSERT |
| P08 编辑 | 写 Seats | UPDATE |
| P08 删除 | 先读 Reservations（检查）→ 写 Seats | DELETE（需先检查关联） |
| P08 切换状态 | 写 Seats | UPDATE Status |
| P09 统计 | 读 Seats + Reservations | 4 条 COUNT / GROUP BY 查询 |
| P06 退出 | 无 | 仅清除 Session |

### 3.5 数据库变化

| 操作 | 表 | 变化 |
|------|-----|------|
| 标记完成 | Reservations | Status: 0→1 |
| 新增座位 | Seats | 新增 1 行 (Status 默认 0) |
| 编辑座位 | Seats | 更新指定字段 |
| 删除座位 | Seats | 删除 1 行（前提：无关联预约） |
| 切换状态 | Seats | Status: 0↔2 |
| 查看统计 | 无 | 只读不写 |

### 3.6 状态流转

```
预约状态（标记完成）：
  0(已预约) ──→ 1(已完成)   [管理员标记]

座位状态（管理端操作）：
  0(空闲)     ──→ 2(维护中) [管理员切换]
  2(维护中)   ──→ 0(空闲)   [管理员切换]
  1(已预约)   ──→ ✗ 不允许   [管理员不可手动切换已预约座位]
```

### 3.7 异常与边界情况

| 编号 | 场景 | 触发条件 | 处理方式 | 提示文案 |
|------|------|----------|----------|----------|
| E11 | 登录失败 | 用户名或密码错误 | 返回登录页，显示错误 | "用户名或密码错误" |
| E12 | 未登录访问后台 | Session 中无 IsAdmin | AdminOnly Filter 拦截，302 跳转到 /Admin/Login | — |
| E13 | 标记不存在的预约 | id 无效 | Service 返回 false | "预约记录不存在" |
| E14 | 标记已取消/已完成的预约 | Status 不为 0 | Service 返回 false | "当前状态不允许标记完成" |
| E15 | 删除有关联的座位 | 该 SeatId 在 Reservations 中有记录 | Service 返回 false | "该座位有关联预约记录，无法删除" |
| E16 | 切换已预约座位状态 | 当前 Status=1 | Service 返回 false | "已预约状态的座位不可手动切换" |
| E17 | 新增座位空字段 | SeatNumber/Area/Floor 为空 | ModelState 验证 | "请填写完整信息" |
| E18 | Session 过期 | 长时间无操作 | AdminOnly Filter 拦截，跳转登录页 | — |

## 5. 默认补足链路 / 当前假设

以前序文档已明确的用户主链路（P01→P05）和管理主链路（P06→P07/P08/P09）为准。以下为本次展开时的补充决策：

### 5.1 默认补足链路

| 补足项 | 选择 | 依据 |
|--------|------|------|
| 取消预约作为 P05 的子操作而非独立页面 | 用户流程更短 | 前序文档页面树中取消是 P05 内的一个按钮操作 |
| 标记完成作为 P07 的子操作而非独立页面 | 同上 | 管理端表格行内的按钮操作 |
| 座位 CRUD 均通过 P08 页面（非独立页面）| 减少页面数 | 新增/编辑使用 Modal 弹窗，删除使用确认弹窗 |
| 在 P04 中合并座位详情中的提交流程 | 两条路径均可 | 方案 A：P03 直接提交；方案 B：P03→P04 确认后再提交。以 PRD-Lite 为准，保留 P04 确认环节 |

### 5.2 当前假设

| 假设 | 说明 |
|------|------|
| 用户 Cookie 不会伪造 | 体验账号体系不设防，不校验身份真实性 |
| 管理员仅一个固定账号 | 不提供注册和修改密码功能 |
| 事务不会死锁 | 课堂数据量极小，不涉及死锁处理 |
| 预约日期不能是过去日期 | Service 层加 `date < DateTime.Today` 校验 |
| 所有 POST 操作不做防 CSRF 处理 | 课堂项目简化，正式项目需加 `[ValidateAntiForgeryToken]` |
| 不记录操作日志 | 管理员标记完成、取消等操作无审计追踪 |
